79`t`t摘要
80`ta3`t随着人工智能技术和在线教育技术的快速发展，传统编程学习方式在知识组织、问题答疑和个性化指导方面逐渐暴露出不足。现有编程学习平台普遍存在知识呈现碎片化、答疑缺乏上下文支撑、学习路径缺少个性化规划等问题，难以满足学习者系统化、高效率的学习需求。为解决上述问题，本文设计并实现了一套基于 Vue3、Spring Boot、Redis、知识图谱与大语言模型（LLM）的智能编程学习助手系统。
81`ta3`t本系统采用前后端分离架构，前端基于 Vue3、Vite 和 Element Plus 实现交互式界面，后端基于 Spring Boot、Spring Security 和 MyBatis-Plus 实现业务逻辑与权限控制，并结合 MySQL、Redis、Neo4j 和 Elasticsearch 构建多源数据支撑体系。其中，MySQL 用于存储用户、会话、代码片段、学习路径与学习记录等结构化数据，Redis 用于热点问答缓存与上下文缓存，Neo4j 用于构建编程知识图谱，Elasticsearch 用于知识文档与知识分块的向量检索。系统在智能问答模块中融合知识文档检索与图谱关系检索，将检索结果作为上下文提供给大语言模型，从而提升回答的准确性与可解释性。
82`ta3`t系统实现了用户注册登录、智能问答、知识图谱导航、个性化学习路径生成、学习看台、练与测、代码片段管理、学习进度跟踪、知识文档管理和学习报告导出等主要功能。其中，学习路径模块不仅支持基于知识图谱依赖关系生成学习节点，还可为每个节点自动关联推荐文档与推荐视频；学习看台模块支持本地视频检索、联网视频检索、在线资源导入、收藏管理与观看历史记录；练与测模块支持练习生成、考试组卷、自动计时交卷、评分分级以及错题沉淀，形成“路径规划—资源学习—练习测评—复盘优化”的学习闭环。学习路径模块可基于知识图谱依赖关系生成学习节点，在图谱覆盖不足时通过大语言模型进行补充；学习进度模块通过对用户行为数据的统计分析，生成仪表盘、热力图、雷达图、智能洞察及 PDF 学习报告，帮助学习者形成闭环学习过程。最后，系统采用 Docker Compose 实现多容器部署，提高了部署效率和环境一致性。
83`ta3`t本文围绕系统需求分析、总体架构设计、核心模块实现与系统测试展开研究。实践结果表明，该系统能够较好地整合知识图谱、检索增强生成与学习行为分析能力，为编程学习提供较为智能、系统和个性化的辅助支持。

85`t`tAbstract
86`ta3`tWith the rapid development of artificial intelligence and online education technologies, traditional programming learning methods have gradually shown limitations in knowledge organization, question answering, and personalized guidance. Existing programming learning platforms generally suffer from fragmented knowledge presentation, weak contextual support in question answering, and insufficient personalized learning path planning, making it difficult to meet learners’ needs for systematic and efficient study. To address these issues, this thesis designs and implements an intelligent programming learning assistant system based on Vue3, Spring Boot, Redis, knowledge graph, and large language model (LLM).
87`ta3`tThe system adopts a front-end and back-end separated architecture. The front end is developed with Vue3, Vite, and Element Plus to provide an interactive user interface, while the back end is implemented with Spring Boot, Spring Security, and MyBatis-Plus to support business logic and access control. In addition, MySQL, Redis, Neo4j, and Elasticsearch are integrated to build a multi-source data support system. MySQL is used to store structured data such as users, conversations, code snippets, learning paths, and learning records. Redis is used for hot question-answer caching and context caching. Neo4j is applied to construct the programming knowledge graph. Elasticsearch is used for vector retrieval of knowledge documents and document chunks. In the intelligent question-answering module, both document retrieval and graph relation retrieval are combined, and the retrieved results are provided to the large language model as context, thereby improving the accuracy and interpretability of generated answers.
88`ta3`tThe system implements key functionalities such as user registration and login, intelligent Q&A, knowledge graph navigation, personalized learning path generation, learning dashboard, practice and assessment, code snippet management, learning progress tracking, knowledge document management, and learning report export. The learning path module not only generates learning nodes based on knowledge graph dependencies but also automatically associates each node with recommended documents and videos. The learning dashboard module supports local video search, online video search, resource import, bookmark management, and viewing history. The practice and assessment module enables exercise generation, exam question compilation, automatic timed submission, scoring grading, and error question accumulation, forming a learning loop of "path planning—resource learning—practice assessment—review and optimization." The learning path module can generate learning nodes based on knowledge graph dependencies and supplement them with large language models when graph coverage is insufficient. The learning progress module analyzes user behavior data to generate dashboards, heatmaps, radar charts, intelligent insights, and PDF learning reports, aiding learners in forming a closed-loop learning process. Finally, the system employs Docker Compose for multi-container deployment, enhancing deployment efficiency and environmental consistency.
89`ta3`tThis thesis focuses on system requirement analysis, overall architecture design, core module implementation, and system testing. The practical results show that the system can effectively integrate knowledge graph, retrieval-augmented generation, and learning behavior analysis, thus providing intelligent, systematic, and personalized support for programming learning.

187`t1`t第 1 章 引言
188`ta3`t本章主要介绍智能编程助手的课题背景与意义。其中主要内容包括分析国内外的编程初学者学习中面临的问题和难点，同时针对现有问题开发一个智能编程助手。
189`t2`t课题背景与意义
190`ta3`t在数字化转型不断推进的背景下，编程能力已经成为软件工程、计算机科学乃至跨学科领域的重要基础能力。与此同时，编程学习者的来源更加多样，既包括高校计算机专业学生，也包括自学者、转岗人员以及初级开发者。传统编程学习方式通常依赖教材、课堂讲授和搜索引擎检索，虽然能够提供一定的知识支持，但在知识组织、答疑效率和个性化指导方面存在明显局限。
191`ta3`t首先，现有编程学习资源普遍呈现碎片化特征。学习者往往通过文档、博客、问答平台和视频课程获取知识，这些资源虽然丰富，但缺少统一的结构化组织方式，导致学习者容易停留在零散知识点层面，难以形成系统化的知识体系。其次，传统在线答疑方式多数依赖关键词匹配或人工解答，响应效率有限，对上下文理解不足，也难以结合相关知识点进行延伸说明。再次，不同学习者的基础水平、学习目标和学习习惯差异较大，而很多平台提供的学习内容与推荐路径较为固定，难以满足个性化需求。
192`ta3`t近年来，大语言模型在自然语言理解、文本生成和代码辅助方面展现出较强能力，为智能化编程学习支持提供了新的技术路径。学习者可以通过自然语言向系统提问，系统能够返回较完整的解释、示例代码和实现思路，这显著提升了交互效率。然而，大语言模型也存在知识来源不透明、事实性错误和回答可解释性不足等问题，特别是在专业技术场景下，如果缺少外部知识支撑，模型容易出现“幻觉”或上下文脱节现象。
193`ta3`t知识图谱能够以“实体—关系—实体”的形式组织知识，描述编程知识点之间的依赖、包含和关联关系，从而为学习者提供结构化、可视化的知识体系支持。将知识图谱与大语言模型结合，一方面可以为模型提供更加明确的知识上下文，增强回答的可靠性与逻辑性；另一方面也可以通过图谱导航帮助学习者理解知识点之间的前后依赖关系，缓解编程学习中的碎片化问题。与此同时，检索增强生成技术通过在生成答案前引入相关知识文档检索结果，可以进一步提升智能问答在垂直领域中的准确率和可控性。
194`ta3`t基于此，设计并实现一套融合前后端分离架构、知识图谱、检索增强生成与大语言模型的智能编程学习助手，具有较强的现实意义和应用价值。一方面，该系统能够为学习者提供集问答、图谱导航、学习路径规划、代码片段管理和学习分析于一体的综合学习平台；另一方面，也为知识图谱与大语言模型在教育场景中的协同应用提供了可落地的系统实现案例。
195`t2`t国内外研究现状
196`ta3`t目前，国内外围绕智能编程学习、知识图谱辅助学习以及大语言模型增强问答等方向已开展了较多研究与实践。
197`ta3`t在国外，GitHub Copilot、ChatGPT 等工具已经在代码补全、问题解答和编程辅助方面展现出较强能力。这类工具能够有效提高开发效率，也能够在一定程度上支持学习者理解代码实现方式。但从学习支持角度来看，其主要侧重于即时生成与答疑，对知识体系的结构化组织相对不足，无法直接帮助学习者建立系统的知识框架。与此同时，一些在线编程学习平台，如 Codecademy、Coursera 和 LeetCode，更注重课程体系和练习机制，但其智能交互和个性化知识解释能力相对有限。
198`ta3`t在国内，编程学习平台和智能问答系统也取得了一定发展。一些研究聚焦于基于知识图谱的教育资源推荐，通过构建知识点关联关系来分析学习路径和薄弱环节；另一些研究则关注将自然语言处理方法应用于自动答疑与学习分析中。然而，很多系统仍以关键词检索、规则匹配或模板化回答为主，缺乏大语言模型支持下的自然语言深度交互能力。即使引入大语言模型，也常常停留在简单问答层面，没有与知识图谱、向量检索和学习行为分析形成完整协同。
199`ta3`t在知识图谱与大语言模型融合方面，当前研究主要包括两个方向：一是通过知识图谱增强大语言模型，在推理或生成阶段提供结构化知识支撑，以减轻幻觉问题并增强可解释性；二是利用大语言模型辅助知识抽取、图谱构建和知识更新，以降低人工维护成本。与此同时，检索增强生成技术已成为垂直领域智能问答的重要实现路径，其核心思想是在模型生成前，从外部知识库检索相关信息作为上下文输入，从而提升回答质量。该技术已经在教育、医疗、法律等领域得到广泛关注。
200`ta3`t总体来看，现有研究虽然为智能编程学习系统的发展提供了重要基础，但仍存在一些不足：第一，部分系统缺少面向编程学习场景的知识体系可视化能力；第二，部分系统缺少对学习过程和学习行为的持续记录与分析；第三，知识图谱、检索增强生成与大语言模型往往作为独立技术使用，缺少统一系统中的深度协同。因此，开发一套面向编程学习场景、同时整合知识图谱、文档检索、大语言模型问答与学习行为分析的系统，具有较强的研究与实践意义。

218`ta3`t此外，系统在智能问答实现上并非单纯依赖大语言模型，而是结合知识文档检索、知识图谱节点检索和关系检索构造上下文，形成了较为完整的检索增强问答流程。对于学习路径生成模块，系统还设计了“图谱优先、模型补充”的方案，使其在知识图谱覆盖不足时仍能生成可用路径。因此，从架构设计到模块落地，本系统在技术上具备实现条件。

221`ta3`t系统在交互流程上遵循“单用户学习闭环”的设计思路，功能边界较清晰。智能问答、图谱导航、学习路径和学习数据分析等模块彼此独立但又能够形成联动，有利于用户在学习过程中持续使用。前端还实现了仪表盘、热力图、雷达图、图谱详情面板和知识文档详情等界面，增强了系统的可操作性与可读性。因此，从运行与使用角度看，系统具备较好的可行性。
222`ta6`t法律可行性分析
223`ta3`t本系统主要用于编程学习辅助，涉及的核心数据包括用户账号信息、学习记录、问答内容和上传头像等。为保证系统安全性，后端采用 Spring Security 与 JWT 鉴权机制对接口进行统一访问控制，未登录用户无法直接访问业务接口。对于用户密码，系统采用加密方式存储；对于头像上传等文件操作，也进行了类型和大小限制，以降低恶意上传带来的风险。
224`ta3`t在数据使用方面，系统主要处理用户主动输入的学习相关数据，不涉及大规模敏感个人隐私信息的采集与处理，整体风险可控。在引用外部知识和模型能力时，系统通过本地知识库、图数据库和缓存机制进行支撑，不直接暴露底层服务细节。总体而言，本系统在安全控制和数据使用方面具备基本的法律与合规可行性。
225`t2`t方案比选
226`ta3`t针对智能编程学习助手的实现目标，可以考虑以下几种方案。
227`t3`t2.2.1 方案一：传统问答式学习系统
228`ta3`t该方案以常规 Web 系统为基础，主要实现用户登录、问题提交和答案展示等功能，问答逻辑主要依赖大语言模型直接生成，不引入知识图谱和检索增强机制。该方案结构简单、开发成本较低，能够较快完成基础问答功能。
229`ta3`t但其缺点也较为明显。首先，回答质量强依赖模型本身，容易出现事实性偏差；其次，无法为学习者提供系统化知识结构展示；再次，难以支持学习路径规划和知识点关联导航。因此，该方案虽然实现难度低，但不符合本课题“智能化、系统化、个性化”的总体目标。
230`t3`t2.2.2 方案二：基于关键词检索的学习平台
231`ta3`t该方案在基础 Web 平台上增加知识文档库，通过关键词匹配从知识库中检索相关内容，再将检索结果返回给用户。这种方案相较于纯生成式问答更可控，也便于系统积累领域知识内容。
232`ta3`t然而，单纯基于关键词检索的方式对语义理解能力较弱，面对表达方式多样、上下文复杂的提问时，检索效果容易下降。同时，该方案仍然缺少知识点间关系表达能力，不利于学习者从整体上理解编程知识体系。因此，该方案虽然具有一定实用性，但在智能性和系统性上仍有不足。
233`t3`t2.2.3 方案三：知识图谱与检索增强生成协同方案
234`ta3`t该方案在前后端分离架构基础上，综合引入知识图谱、知识文档库、向量检索和大语言模型。系统在用户提问后，不直接让模型独立生成答案，而是先通过知识文档检索和图谱节点关系检索获取相关上下文，再将这些信息提供给模型进行回答生成。同时，系统通过知识图谱可视化展示知识点之间的依赖和关联关系，并基于知识图谱或模型生成学习路径，进一步结合学习行为数据实现学习分析。
235`ta3`t相较于前两种方案，该方案既保留了大语言模型的自然语言交互能力，又增强了知识支撑与结果可解释性，能够更好满足课题目标。虽然该方案涉及的技术模块较多，实现复杂度更高，但从系统完整性与研究价值来看，更适合作为本课题的最终方案。
236`ta3`t综上比较，本文最终采用方案三，即基于 Vue3、Spring Boot、Redis、知识图谱与大语言模型协同的智能编程学习助手方案。
237`t2`t本文方案
238`ta3`t为高性能缓存，加速高频数据的读取与写入，有效缓解数据库压力。
239`ta3`t本文系统采用前后端分离架构，整体可分为前端展示层、后端业务层、数据存储层和智能服务支撑层四个部分。
240`ta3`t前端展示层基于 Vue3、Vite 和 Element Plus 构建，负责实现用户登录、聊天界面、知识图谱浏览、学习路径展示、代码片段管理、知识文档管理和学习进度分析等页面，并通过 Axios 与后端接口进行通信。
241`ta3`t后端业务层基于 Spring Boot 搭建，结合 Spring Security 实现接口访问控制，使用 MyBatis-Plus 完成关系型数据读写。后端对外提供认证管理、智能问答、图谱服务、学习路径、代码片段、知识文档和学习进度等业务接口。
242`ta3`t数据存储层采用多数据库协同设计。其中，MySQL 用于保存用户、会话、消息、反馈、学习路径、学习节点、学习记录、知识文档与代码片段等结构化数据；Redis 用于缓存热点问答结果和会话上下文；Neo4j 用于存储编程知识点及其关系；Elasticsearch 用于存储知识向量索引与知识分块索引，支持语义检索和混合检索。
243`ta3`t智能服务支撑层主要由大语言模型调用、检索增强生成流程和本地向量化处理构成。系统通过知识文档检索、图谱节点检索和关系检索构建上下文，再调用大语言模型生成回答；在学习路径模块中，系统优先根据知识图谱依赖关系生成路径，若目标知识在图谱中覆盖不足，则通过模型生成补充路径节点。
244`t2`t方案中拟考虑的安全、法律制约因素
245`ta3`t在系统方案设计过程中，安全性是不可忽视的重要因素。为了保证系统在编程学习场景中的稳定运行与数据可控使用，本文从身份认证、数据保护、系统调用和多源存储四个方面进行安全设计考虑。
246`ta3`t第一，身份认证与接口访问安全。系统采用 JWT 作为无状态认证机制，并结合 Spring Security 对 `/api/**` 接口进行统一访问控制。除注册、登录等必要开放接口外，其余核心业务接口均需在认证通过后方可访问，从而降低未授权访问风险。
247`ta3`t第二，用户数据安全。系统对密码进行加密存储，避免明文保存；对头像上传功能进行文件类型与大小校验，降低恶意文件上传风险；对个人资料、学习记录、会话消息等数据按照用户维度进行隔离访问，防止越权读取。
248`ta3`t第三，系统调用安全。智能问答功能涉及大语言模型调用、知识检索和上下文拼接等复杂流程。为了提高系统健壮性，后端采用统一异常处理与标准化响应封装机制，对输入异常、业务异常和系统异常进行集中管理，避免将不必要的底层错误信息直接暴露给前端。
249`ta3`t第四，缓存与多数据源协同安全。Redis、MySQL、Neo4j 和 Elasticsearch 在系统中承担不同职责。为避免跨层混用导致的数据污染与状态异常，系统在业务实现上保持数据访问边界清晰，通过业务服务层统一调度各类数据资源，从而降低错误写入与不一致问题带来的影响。
250`t2`t方案对环境及可持续发展方面的影响
251`ta3`t本系统采用在线化、数字化方式支持编程学习，将问答、知识导航、路径规划、代码沉淀和学习复盘整合到统一平台中，减少了纸质资料整理、线下重复沟通和多平台频繁切换的资源浪费，具有一定的绿色化特征。
252`ta3`t同时，系统通过 Docker Compose 实现多服务统一部署，减少人工配置环境的重复劳动，提高资源利用效率和环境复现能力。系统在架构上采用模块化、分层化设计，后续可以在原有基础上继续扩展知识文档、知识节点、模型能力和推荐策略，而不需要整体推倒重建。这种渐进式演进方式有利于降低后续维护和改造成本，也符合现代软件系统可持续迭代的发展思路。
253`t2`t本章小结
254`ta3`t本章从经济、技术、运行和法律安全等方面对智能编程学习助手系统进行了可行性分析，并对三种典型实现方案进行了比较。最终，本文选择了知识图谱、检索增强生成与大语言模型协同的总体方案，并结合前后端分离架构与多数据源设计，形成了本系统的总体技术路线。该方案能够较好满足智能问答、知识导航、学习路径规划和学习行为分析等核心需求，为后续需求分析、系统设计与功能实现奠定了基础。

286`t2`t3.2 功能需求分析
287`ta3`t结合当前项目实现情况，本文将系统功能需求归纳为七个主要模块。
288`t3`t3.2.1 用户认证与个人信息模块
289`ta3`t系统应支持用户注册、登录、退出和当前登录用户信息查询，并支持头像上传与个人资料摘要展示。该模块是其他业务功能的访问基础，除注册和登录等开放接口外，系统主要业务接口均应在用户完成身份验证后方可访问。
290`ta3`t在实现上，用户认证模块不仅承担账户管理功能，还承担学习数据入口控制作用，确保会话、路径、代码片段和学习分析等数据能够按照用户身份进行隔离。
291`t3`t3.2.2 智能问答模块
292`ta3`t系统应支持用户通过自然语言输入编程相关问题，并完成问题接收、会话创建、消息保存、知识检索、上下文构造和回答生成等处理流程。回答应支持流式返回，以提升用户交互体验。
293`ta3`t除基础问答能力外，系统还应支持历史会话查看、会话删除、回答反馈、来源信息展示和检索指标查看等功能。该模块是整个系统最核心的功能模块，也是其他知识支撑模块的主要调用入口。
294`t3`t3.2.3 知识图谱导航模块
295`ta3`t系统应支持展示编程知识图谱总体结构，并提供知识节点搜索、节点详情查看、邻居节点展开、关系查看、图谱健康度分析和节点关联文档查看等功能。该模块的目标是帮助用户从结构层面理解知识点之间的前置依赖、包含关系和相关联系，而不仅仅停留在零散问题的回答层面。
296`t3`t3.2.4 学习路径模块
297`ta3`t系统应支持根据用户输入的学习目标生成个性化学习路径。若知识图谱中存在相关依赖结构，则系统优先基于图谱关系生成推荐路径；若图谱中覆盖不足，则由模型生成补充节点。系统还应支持路径查看、路径删除和学习节点状态更新等功能。
298`ta3`t该模块的重点在于把抽象的学习目标转化为有顺序、有阶段、有进度状态的学习任务序列，增强系统对持续学习的支撑能力。
299`t3`t3.2.5 代码片段管理模块
300`ta3`t系统应支持代码片段的新建、编辑、删除、列表查询、关键字搜索、语言筛选、标签筛选、使用次数统计、导入导出以及基于当前会话推荐片段等功能。该模块主要服务于用户在学习过程中对代码知识的沉淀、积累和复用。
301`ta3`t相比单纯保存代码，系统还应具备在问答上下文中进行相关片段推荐的能力，从而增强问答与知识沉淀之间的联动性。
302`t3`t3.2.6 学习进度跟踪与分析模块
303`ta3`t系统应记录用户在问答、图谱探索、代码保存、路径学习和反馈等过程中的学习行为，并基于这些数据生成仪表盘、热力图、雷达图、智能洞察、薄弱项分析和周计划等可视化结果。同时，系统还应支持学习报告 PDF 导出，以便用户进行阶段性复盘。
304`ta3`t该模块的意义在于将用户分散的学习行为转化为可观察、可分析、可复盘的学习数据，从而使系统具备持续性学习支持能力。
305`t3`t3.2.7 知识文档管理模块
306`ta3`t系统应支持知识文档的新增、查询、更新和删除等管理能力，并支持查看文档与图谱节点之间的关联关系。该模块既服务于后台知识维护，也为检索增强问答提供知识来源支撑，是整个智能问答系统的重要基础设施。
307`t3`t3.2.8 学习看台模块
308`ta3`t学习看台模块用于承载视频化学习场景，支持本地学习视频检索与联网视频检索双入口。用户可对视频进行播放、收藏、历史回看与在线导入，系统需记录用户观看时长、最近观看时间和完成状态，并以“收藏夹+观看历史”形式提供复用入口。该模块与学习路径和智能问答联动，支持从路径节点推荐视频直接跳转学习看台，实现资源即点即学。
309`t3`t3.2.9 练与测模块
310`ta3`t练与测模块用于学习成果巩固与阶段评估，提供练习模式与测评模式。练习模式支持按学习路径生成题目并逐题即时判定；测评模式支持题量与时长配置、倒计时管理、交卷评分和等级评定。系统需支持错题自动沉淀、题目收藏、历史记录回看与结果可视化，帮助用户形成“学后即练、练后即测、测后复盘”的闭环。

585`ta3`t图3- 9 领域模型图图3- 9 领域模型图根据系统当前实现，可以抽象出以下主要领域对象，领域模型图如图3- 9所示：

588`ta3`t1. 用户（User）：保存用户名、密码、邮箱、头像、学习等级等基础信息。
589`ta3`t2. 会话（Conversation）：表示用户与系统之间的一次问答会话，包含标题、所属用户和时间信息等。
590`ta3`t3. 消息（Message）：表示会话中的具体消息内容，包括用户提问和系统回答，并可附带来源信息。
591`ta3`t4. 反馈（Feedback）：记录用户对系统回答的评价结果，如有用、无用等。
592`ta3`t5. 代码片段（CodeSnippet）：保存用户沉淀的代码内容、标题、语言、标签、描述和使用次数等信息。
593`ta3`t6. 学习路径（LearningPath）：表示针对某一学习目标生成的学习路线。
594`ta3`t7. 学习节点（LearningNode）：表示学习路径中的具体节点，记录知识点标识、顺序和学习状态。
595`ta3`t8. 学习记录（LearningRecord）：记录用户行为，如问答、图谱探索、路径学习、代码保存和反馈等，为学习分析提供数据支撑。
596`ta3`t9. 知识文档（KnowledgeDocument）：保存知识库中的文档内容、标题、类别与检索相关信息。
597`ta3`t10. 知识分块（KnowledgeChunk）：将知识文档切分为更细粒度内容单元，以提升检索准确度。
598`ta3`t11. 知识节点（Knowledge）：存储于 Neo4j 中，表示编程知识图谱中的知识实体。
599`ta3`t12. 知识关系（Relation）：表示知识节点之间的依赖、包含和关联关系。
600`ta3`t这些领域对象共同构成系统的核心业务模型：用户通过会话与系统交互，消息与反馈形成问答闭环；用户通过学习路径、学习节点和学习记录形成学习过程管理；知识文档、知识分块、图谱节点和图谱关系共同构成知识支撑层。
601`ta3`t13.学习视频（LearningVideo）：存储视频标题、平台、链接、封面、时长、标签等信息；

683`t3`t4.4.2 Neo4j 图谱设计
