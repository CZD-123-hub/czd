# 智能编程学习助手项目检查清单（学生汇报版）

这份清单可以按“我向老师介绍项目”的方式来使用。重点不是只说页面有什么，而是说明每个功能模块背后的实现思路、使用技术、关键代码位置、数据存储方式和现场演示点。

## 一、项目整体介绍

本项目是一个面向编程学习者的智能编程学习助手系统，主要目标是帮助学生在学习编程过程中完成智能问答、知识检索、知识图谱浏览、学习路径规划、代码片段管理和学习效果分析。

系统不是简单调用大模型接口，而是在大模型问答的基础上加入了知识文档检索、Elasticsearch 向量检索、Neo4j 知识图谱关系增强、Redis 缓存优化和学习行为分析，使回答内容更贴近本系统知识库，也能体现编程学习过程中的个性化辅助能力。

## 二、项目技术栈总览

| 层次 | 使用技术 | 作用 |
|---|---|---|
| 前端展示层 | Vue 3、Vite、TypeScript、Element Plus、Pinia、Axios | 实现登录、问答、图谱、学习路径、代码片段、学习看板等页面 |
| 可视化层 | ECharts、图谱渲染组件、雷达图、热力图 | 展示知识图谱、学习热力图、知识掌握雷达图等 |
| 后端业务层 | Spring Boot 3、Spring Security、JWT、MyBatis-Plus | 实现接口服务、用户认证、业务逻辑和数据库访问 |
| 智能问答层 | RAG、SSE 流式输出、Embedding、QueryRewrite、LLM 接口 | 实现检索增强生成、流式回答和上下文问答 |
| 数据存储层 | MySQL、Redis、Neo4j、Elasticsearch | 分别保存业务数据、缓存、图谱关系和向量索引 |
| 部署运行 | Docker Compose、Maven、Node.js、Nginx | 统一启动数据库、中间件、前后端服务 |

### 实现方式

智能问答模块采用 RAG 检索增强生成方式。用户提出问题后，系统不会直接把问题发给大模型，而是先根据问题进行查询扩展，然后从知识文档、知识块和知识图谱中检索相关内容，再把检索结果组合成 Prompt 输入给大模型。大模型生成答案后，后端通过 SSE 流式推送给前端，前端逐字展示回答内容。

### 核心处理流程

| 步骤 | 说明 |
|---|---|
| 1 | 用户在前端输入问题并发送 |
| 2 | 后端保存用户消息和会话信息 |
| 3 | QueryRewrite 对问题进行查询词扩展 |
| 4 | Elasticsearch 优先检索相关知识块和知识文档 |
| 5 | 如果 Elasticsearch 不可用，使用本地 embedding 相似度和关键词检索兜底 |
| 6 | Neo4j 检索相关知识节点和关系，补充图谱依据 |
| 7 | 后端将文档、知识块、节点、关系组合为上下文 Prompt |
| 8 | 调用大模型接口生成回答 |
| 9 | 通过 SSE 把回答内容流式返回前端 |
| 10 | 保存助手回答、来源信息和反馈记录 |
| 11 | Redis 缓存问答结果和会话上下文 |

### 使用技术

| 技术 | 作用 |
|---|---|
| RAG | 根据本地知识库增强大模型回答 |
| SSE | 实现浏览器端实时流式接收回答 |
| Elasticsearch | 对知识文档和知识块进行向量检索 |
| Embedding | 将文本转换为向量，用于语义相似度计算 |
| QueryRewrite | 扩展用户问题中的关键词，提高召回率 |
| Neo4j | 提供知识节点和关系依据 |
| Redis | 缓存问答结果和会话上下文 |
| MySQL | 保存会话、消息、反馈等业务数据 |

| Markdown/代码块渲染 | 前端展示知识内容 |
| MyBatis-Plus | 数据库访问 |

## 八、如果老师追问，可以这样回答

### 1. 这个项目的核心业务是什么？

核心业务是围绕编程学习场景提供智能辅助，包括智能问答、知识检索、知识图谱导航、学习路径规划、代码片段管理和学习效果分析。其中智能问答是核心入口，其他模块为问答和学习过程提供知识支撑、代码支撑和数据反馈。

### 2. 智能问答和普通 ChatGPT 有什么区别？

普通 ChatGPT 主要依赖模型自身知识，而本系统会先检索本地知识文档、知识块和知识图谱关系，再把这些内容和用户问题一起提交给大模型。因此回答更贴近本系统的学习资料，并且可以展示来源依据。

### 3. Elasticsearch 检索增强体现在哪里？

系统会把知识文档和知识块转换为 embedding 向量并写入 Elasticsearch。用户提问时，系统使用问题向量在 Elasticsearch 中查找语义相似的知识块，然后把这些内容作为上下文给大模型。这样可以提高相关资料召回率，减少用户自己翻资料的时间。

### 4. 如果 Elasticsearch 不可用，系统会崩溃吗？

不会。系统实现了本地检索兜底逻辑，当 Elasticsearch 检索失败时，会使用本地 embedding 相似度和关键词混合检索继续查找知识内容，保证智能问答功能仍然可用。

### 5. Neo4j 在系统里有什么实际作用？

Neo4j 主要用于保存知识点之间的关系。它有三个作用：第一，支持知识图谱页面的可视化展示；第二，在智能问答中提供节点依据和关系依据；第三，在学习路径生成时根据知识点依赖关系组织学习顺序。

### 6. Redis 在系统里有什么实际作用？

Redis 主要用于缓存问答结果、会话上下文和代码推荐反馈。这样可以减少重复查询数据库和重复调用大模型，提高系统响应速度。

### 7. 学习报告有什么依据？

学习报告基于用户在系统中的真实行为生成，包括问答次数、学习路径节点完成情况、代码片段使用情况、知识图谱探索情况和反馈情况。系统根据这些数据计算学习健康度、待提升领域和本周行动计划。

### 8. 项目的创新点是什么？

项目的创新点主要体现在把智能问答、知识图谱、向量检索和学习行为分析结合起来，形成一个编程学习闭环。用户不仅能问问题，还能看到依据、顺着知识图谱探索、生成学习路径、沉淀代码片段，并通过学习看板观察自己的学习效果。

## 十、汇报时可以使用的一段总结

我的项目是一个面向编程学习场景的智能编程学习助手系统。系统采用前后端分离架构，前端使用 Vue 3 和 TypeScript，后端使用 Spring Boot，并结合 MySQL、Redis、Neo4j 和 Elasticsearch 实现数据存储、缓存、图谱关系和向量检索。核心智能问答模块采用 RAG 检索增强生成方式，系统会先从知识文档、知识块和知识图谱中检索相关内容，再组合 Prompt 调用大模型生成回答，并通过 SSE 实现流式输出。同时，系统还提供知识图谱探索、学习路径生成、代码片段管理和学习看板报告等功能，形成从提问、学习、实践到分析反馈的完整编程学习闭环。
