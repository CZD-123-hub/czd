                                        摘要

      钢铁行业是国民经济的重要支柱，其智能化转型对提升效率和安全性至关重
要。高炉炼铁作为核心环节，操作复杂且依赖经验，需要智能决策的支持。但传
统问答系统专业性不足，大模型存在幻觉和知识滞后问题，且钢铁生产数据具有
隐私性。为此，本文基于检索增强生成（RAG）技术和本地知识库，开发了高炉
智能问答系统来解决这些问题。

      在技术实现层面，本文首先提出改进的文本分块方法（Chinese Recursive Text
Splitter，CRTS），优化了中文文档的分块效果，并且采用了基于 BGE-Large-zh
模型的文本向量化处理，为高炉领域知识构建了精准的语义表征空间。其次，在
检索阶段设计了基于加权混合与重排序的检索算法，结合 BM25 检索与 FAISS 向
量检索的优势，使 NDCG@10 达到 50.3，提高了检索精度。最后问答生成阶段，
通过优化提示词模板并集成 DeepSeek 大模型，确保答案的专业性与可解释性。

      在完成关键技术优化基础上，本文实现了从理论创新到工程应用的完整转化。
系统采用 SQL Server 关系数据库和 FAISS 向量数据库来存储高炉生产数据和知识
库相关数据。并且通过 LangChain 框架整合上述算法与模型，基于 FastAPI 构建
了支持并发查询的高性能后端服务，配合 Streamlit 开发了包含知识库管理、RAG
问答、智能数据库查询等功能的交互界面。实际测试表明，系统显著提升了高炉
操作的智能化水平，不仅解决了传统问答系统的局限性，也为钢铁行业的数字化
转型提供了可靠的技术支持。

      关键词：问答系统；大语言模型；RAG 技术；词嵌入模型；混合检索
                                     Abstract

      The steel industry is a crucial pillar of the national economy, and its intelligent
transformation is essential for improving efficiency and safety. Blast furnace
ironmaking, as a core process, involves complex operations and relies heavily on
experience, necessitating intelligent decision-making support. However, traditional
Q&A systems lack domain-specific expertise, large language models suffer from
hallucinations and outdated knowledge, and steel production data is highly sensitive.
To address these challenges, this paper develops an intelligent Q&A system for blast
furnaces based on Retrieval-Augmented Generation (RAG) technology and a local
knowledge base.

      At the technical implementation level, this research first introduces an enhanced
text segmentation method—the Chinese Recursive Text Splitter (CRTS)—which
optimizes the chunking of Chinese technical documents. The system then employs the
BGE-Large-zh embedding model to create accurate semantic representations of blast
furnace domain knowledge. For the retrieval phase, we design a hybrid algorithm that
combines BM25 keyword search with FAISS vector similarity search, augmented by a
re-ranking mechanism. This approach achieves an NDCG@10 score of 50.3,
significantly improving retrieval precision. In the answer generation stage, we optimize
prompt engineering templates and integrate the DeepSeek large language model to
ensure responses maintain both technical accuracy and interpretability.

      Building on these key technical optimizations, this paper accomplishes a complete
transition from theoretical innovation to engineering application. The implemented
system uses SQL Server for structured data and FAISS for vector storage. Through
LangChain framework orchestration and a FastAPI backend, it efficiently handles
concurrent queries. A Streamlit interface provides complete functionality including
knowledge management, RAG Q&A, and intelligent database querying. Practical tests
confirm the system's effectiveness in enhancing blast furnace operations, overcoming
traditional Q&A limitations while supporting steel industry digital transformation.

      Keywords: Question-answering system; Large Language Model; RAG
technology; Embedding model; Hybrid retrieval

1.1. 研究背景与意义

      钢铁行业作为传统制造业，是国民经济不能轻言忽略的产业，随着世界可持
续发展所提出的新需求、新挑战，钢铁行业也必须进行转型升级。国务院办公厅
下发的《中国制造 2025 发展规划》指出传统制造业，尤其是钢铁行业，需要通过
信息智能化技术实现数字化、网络化、智能化和绿色化转型[1]。钢铁生产的核心是
高炉炼铁，高炉炼铁成本造价、能耗、CO2 排放均占钢铁行业 70%以上，高炉以
及其附属工序设备工艺复杂多样，生产过程中充满不确定性，并且所有变化均发
生在封闭的高炉内部，难以直接监测，因此被视为典型的“黑箱”问题[2]。

      传统的高炉操作主要是依据操作人员的经验知识和操作知识来操作，容易受
到人为因素的影响，导致操作失误，影响生产效率，难以实现长期高效、低耗和
炉况稳定运行[3]。由于煤矿工作人员的安全认知水平有限，以及决策信息的不全面，
仍存在一定的安全隐患。同时，在高炉操作中，操作人员需要及时获取高炉内各
类环境数据，以便做出准确的决策和调整生产过程。但传统的数据库查询方式不
够智能、人性化和定制化。

      近年来人工智能快速发展，智能问答系统应用于各个领域。2022 年 11 月 30
日，OpenAI 推出的 ChatGPT 等大语言模型，凭借其出色的文本理解和生成能力，
推动了人工智能技术与科学研究的深度融合[4]。但是大语言模型（Large Language
Model，LLM）无法完成超出其训练数据范围的内容，在实际应用中大语言模型也
存在幻觉性问题，“幻觉”是大语言模型产生的看似合理但不正确甚至是不符合
逻辑的内容[5]。为了解决该问题，检索增强生成（Retrieval-Augmented Generation，
RAG）技术应运而生。RAG 技术通过检索外部知识库，结合大语言模型的生成能
力，实现了对复杂问题的精准回答[6]。同时，自然语言转 SQL 技术也应运而生，
它能够将用户的自然语言输入转化为 SQL 查询语句，从而使缺乏编程经验的操作
人员也能够通过简单的语言进行高效的数据查询。这项技术不仅大大简化了数据
库查询的操作流程，还提高了系统的可用性和交互性。

      为了满足了钢铁行业转型升级的需求，本文设计并实现了一套基于大语言模
型和 RAG 技术的高炉智能问答系统，能够实现与操作人员的交互式问答与决策辅
助，提升了高炉操作效率与安全性以及优化了数据库查询体验。通过将高炉领域
相关资料与操作数据集成到系统中，利用大语言模型和 RAG 技术，结合提示词工

1.2. 国内外研究现状及发展趋势分析

1.2.1. 问答系统的研究现状及发展趋势

      问答系统属于人工智能领域里比较关键的一种应用，依靠自然语言处理技术
达成对用户问题的自动精准作答，近年来取得了显著进展，当下问答系统的研究
办法主要有微调（Fine-tuning）、提示学习（Prompt Learning）、知识图谱（Knowledge
Graph, KG）和检索增强生成（RAG）这四大类，它们各有特点并且相互补充，一
起推动着问答系统朝着更高效率、更智能化的方向发展。

      随着大语言模型和相关技术的发展，近年来越来越多学者将 RAG 技术应用于
构建各领域的问答系统，以实现高效知识获取与利用，基于 RAG 的问答系统已成
为众多领域的重要发展方向。黄冰[7]构建基于 RAG 的腕足动物知识问答系统，以
交互式问答的方式，帮助学生更好地理解和吸收古生物学专业知识。海佳丽[8]等人
开发的中医药标准知识问答系统，通过对比多种大语言模型并结合数据优化等技
术，在中医文献问题生成数据集和中医药标准问答数据集上，答案相关性和上下
文相关性的各项指标表现出色，有力推动了中医药知识的传播与应用。张文超[9]
等人运用 RAG 技术整合医院现有知识库数据，构建医院 IT 运维知识库问答系统，
该系统上线后访问量持续增长，不仅提升了临床医护人员对信息化运维服务的满
意度，还显著减轻了信息科的运维负担，有效解决了传统知识库使用复杂、检索
困难的问题。鞠炜刚[10]等人提出了一种基于大语言模型和检索增强生成(RAG)的持
续交付智能问答系统构建方法。该方法通过高质量语料处理形成数据集，采用高
效微调技术训练领域大模型，使用改进的向量知识检索并结合提示词工程的多场

1.2.2. 检索增强生成（RAG）技术的研究现状及发展趋势

      大语言模型在自然语言处理任务里有着出色表现，然而在实际运用当中却面
临着如幻觉、知识更新迟缓、答案不透明等诸多挑战。检索增强生成（RAG）技
术应运而生，它在大语言模型生成回答之前，会先从外部知识库检索信息，这样
能显著提升答案准确性，减少幻觉情况[12]。RAG 的优势在于引用知识来源，可让
用户可验证答案准确性，提高对模型输出的信任，助力知识更新和特定领域知识
引入。RAG 把大语言模型的参数化知识和外部非参数化知识库相结合，成为扩展
知识、降低成本、提升质量的一种关键方法，在 RAG 的开发优化过程中，性能评
估是很关键的一点。当前对于 RAG 在不同大语言模型上的影响评估并不充分，评
估结果可能会因检索器不同而有所差异[13]，研究人员制定了核心评估指标：生成
答案的可信度、相关性和上下文相关性。

      在 RAG 的 研 究中 ， 张 艳 萍 等人 [14]提 出 了一 种 多 策 略 检索 增 强 生 成 方法
（MSRAG），用来解决 RAG 技术在军事领域应用时的两个主要问题，分别是混
合检索的可移植性不强以及非必要查询改写造成的语义漂移，MSRAG 方法经过自
适应匹配检索模型、利用文本过滤器提取关键文本块、基于同义词拓展的查询改
写以及提示工程输入生成答案模型等步骤，提高了答案的准确性与相关性。Eric
Melz[15]提出 ARM-RAG 系统，凭借神经信息检索存档小学数学问题的推理链，证
实 了推 理 链存 储 和检 索 对 问题 表 现有 着 积极 影 响。 Akari Asai 等 人 [16]提 出的
Self-RAG 框架，依靠检索和自我反思提升大语言模型生成的质量与真实性，实验
显示其在开放域问答、推理和事实验证任务上比 ChatGPT 和检索增强的 Llama 2
更具优势。Jakub Lála 等人[17]提出的 PaperQA 模型，利用 RAG Agent 在全篇科学

献全文并凭借代理的决策解释答案。在 LitQA 基准测试集上，该模型取得了优秀

      总之，RAG 技术在提升大语言模型的性能、解决幻觉问题以及扩展知识来源

2.1. 大语言模型（LLM）

      LLM 作为自然语言处理领域的前沿技术，近年来得到了广泛的关注与研究，
研究人员在大规模语料库上预训练 Transformer 模型产生了预训练语言模型，这些
模型在文本生成、语言理解、文本分类等多个任务上展现出了强大的能力。LLM
通常指 1B 以上的参数量的语言模型，使用了海量的文本数据进行使用训练，使其
在自然语言理解和生成方面达到了全新的台阶，并产生了小模型不存在的能力，
如上下文学习。

      如图 2.1 所示，2017 年，Transformer 架构问世，凭借自注意力机制革新自然
语言处理。次年，Google 推出 BERT-1，OpenAI 发布 GPT-1，正式开启预训练语
言模型时代。2019 年，模型参数量迎来爆发式增长，OpenAI 的 GPT-2 参数量达
15 亿，同年 Google 发布参数规模 110 亿的 T5 模型。2020 年，拥有 1750 亿参数
的 GPT-3 展示出强大的少样本和零样本学习能力。2022 年 11 月，OpenAI 基于
GPT-3.5 架构推出 ChatGPT，其在通用知识问答中的出色表现引发全球关注，推动
大语言模型进入爆发式增长阶段。2023 年 3 月，OpenAI 发布 GPT-4，性能大幅提
升且具备多模态理解能力，在多项基准考试中成绩优异，超过 88%的应试者。​

2.2. 检索增强生成（RAG）技术

      由于大语言模型依赖预训练数据，它缺乏有些专业领域的的相关知识以及提
供最新动态更新的能力。例如，用户向 ChatGPT 提出某炼钢厂的员工操作规范相
关问题时，由于该文档具有隐私性，并不能被收纳到预训练数据中，就导致查询
内容超出其训练数据的范围，ChatGPT 就会开始出现“幻觉”，自己编出虚假内
容回复给用户。RAG 可以通过外部数据库，弥补这一信息鸿沟。它通过从知识库
中检索与问题相关的内容，并且与用户问题相结合，形成一个全新的提示词，从
而提高 LLMs 生成答案的准确性。如图 2.2 所示，展示了 RAG 的工作流程，突出
其在处理用户查询的时的主要步骤：索引、检索和生成。

2.2.1. 索引
      索引构建是 RAG 技术发挥效能的关键基础，其核心任务是把来源多样、形式

不同的原始数据，整理成能让计算机快速调取的有序结构，在实际操作时，首先
要对各类格式的文档内容，进行全面的清洗与标准化处理。在这个过程中，像特
殊符号、乱码等噪声数据会被去除，冗余的重复信息也会一同清理，以此为后续
处理构建纯净的数据基础。以长篇的技术规范文档为例，为了符合 LLM 对输入文
本长度的限制，并且保证检索的精准度，大多时候会运用传统的固定大小分块
（Fixed-size Chunking）或语义分块（Semantic Chunking）技术，把它拆分成合适
长度的文本块。如图 2.3 所示，传统的固定大小分块虽然实现起来比较简单，但是
大多时候会破坏文本的自然语义边界。而基于语义的分块策略可保持文本的完整
性，不过实现的复杂度比较高，出色的分块策略需要在语义完整性和检索效率之
间找到平衡，一般每个 Chunk 的 token 数量会控制在 512 至 1024 之间，这样能让
LLM 高效处理，又能较好地保留文本的语义连贯性。

型将文本的语义信息嵌入到高维向量空间里，这些向量可表达文本内容，还可以
捕捉文本之间的相似性以及关系，让计算机有效地开展文本检索、分类、聚类等
任务。当前的主流做法是借助基于 Transformer 架构的预训练向量化模型，像
BERT[18]、RoBERTa[19]这类经典模型，以及 OpenAI 推出的专用嵌入模型，都是热
门的选择，这些模型依靠多层自注意力机制，仔细剖析文本 Chunk 的语义内涵，
并且把它映射成高维语义向量。每一个向量维度都包含着特定的语义特征，比如

2.2.2. 检索

      在 RAG 技术里，检索阶段的关键任务是在知识库当中精准地找到和用户查询
最为相关的文本片段，当用户输入查询内容时，系统会先运用预训练的 Embedding
模型把查询文本转变成高维向量表示，接着借助计算该查询向量跟知识库内所有
文本片段向量的距离，给每个候选片段做出相关性评分。当前较为常用的两个计
算向量相似度的模型是余弦相似度以及欧氏距离[20]，为提升检索效率，系统一般
会采用近似最近邻搜索算法（如 HNSW 或 IVF）快速挑选出相似度最高的前 K 个
候选片段，这些候选片段会经历的重排序与筛选，最终选出相关性最高的若干片
段作为生成阶段的上下文输入，整个检索过程要考虑语义相似度，还要兼顾检索
效率以及结果多样性，以此保证输入生成模型的信息准确又全面。
2.2.3. 生成

建质量对最终生成效果起着决定性作用，它要精准传达用户查询意图，还要高效
整合检索到的关键 Chunk，并且借助结构化设计以及明确指令来引导生成模型。把
对话历史融入到 Prompt 中，使系统可支持多轮对话交互，这样就提升了回答的连

特性让基于大语言模型的应用可获取更广泛的数据支持，拓展了应用的功能边界。

      LangChain 的链（Chain）和代理（Agent）机制对复杂应用开发起到了有力的

3.1. 文本分块

      知识库中的文档一般都是长篇知识文档，但是大模型处理会有上下文长度的
限制，同时长文档中各个片段的语义之间可能存在较大的差异，如果当成一个整
体来做知识检索会存在语义的杂揉，应当将长文档切分成更多的小块。文本分块
（Text Chunking），就是将庞大的文本资料分割成一系列更小和更易处理的文本
片段（Chunks）的过程。这些 Chunks 是 RAG 系统中信息处理的基本单元，它们
将被送入 Embedding 模型进行向量化，然后存入向量数据库进行索引，最终服务
于检索环节。文本分块是决定检索质量的关键前置步骤，粗暴或不恰当的分块会
导致信息丢失、上下文割裂、检索效率低下等问题，严重影响 RAG 系统的最终表
现。常见的分块策略有固定大小分块、基于句子的分块、递归字符分块等。

      固定大小分块这是最简单的方法，直接按照固定的字符数或 Token 数来切割
文本，为了缓解在边界处强行切断语义的问题，通常会设置一个“重叠”（Overlap）
大小。重叠部分意味着每个块的末尾会与下一个块的开头有一段重复的内容，但
是这种方法很有可能在句子中间、单词中间（如果按字符切）、代码行中间等不
恰当的地方断开，导致上下文严重割裂。

      基于句子的分块这种策略试图尊重语言的自然边界——句子，它使用句子分
割算法（如基于标点符号.?!，或使用 NLP 库如 NLTK）将文本分割成独立的句子，
然后将一个或多个连续的句子组合成一个 Chunk，使其大小接近目标范围。这种方
法能够更好地保持语义完整性，但缺点是句子长度差异大：有的句子很短，有的
很长，导致 Chunk 大小不均匀，可能会影响后续处理和检索稳定性。

      递归字符分块试图按一个预设的“分隔符”优先级列表来递归地分割文本，
如提供一个分隔符列表，按优先级从高到低尝试分割，优先尝试按\n\n(段落)分割，
如果分割后的块仍然太大，依次尝试按\n(换行符)分割和空格` `分割，最后如果还
太大，就按字符""分割。目标是在保持较大语义块（如段落）的同时，确保最终块
大小不超过限制。

      递归字符文本分割器（Recursive Character Text Splitter，RCTS）是 LangChain
等 框 架 中 常 用 的 一 种 更 智 能 的 策 略 。 如 图 3.1 所 示 ， 先 初 始 化 目 标 块 大 小
（chunk_size）、块间重叠量（chunk_overlap）和优先级分隔符列表（如["\n\n", "\n",
` `, ""]），从最高优先级分隔符（如段落分隔符\n\n）开始对文本进行第一层分割，

                                                                         13
                                              南昌航空大学学士学位论文

生成初始块；然后检查每个块的大小，若发现大于 chunk_size 的块，就自动降级
使用次优先级分隔符（如换行符\n）进行递归分割，此过程循环迭代直至所有块满
足大小要求或分隔符用尽，对于经过全部分隔符尝试后仍超限的顽固块，最终采
用字符级硬截断强制分块，同时在所有相邻块之间按预设的 chunk_overlap 值插入
重叠部分以保持上下文连贯性。整个过程通过分层递归策略实现了语义敏感的自
适应分块，优先在自然边界（段落、句子、词汇）处拆分，仅在必要时才执行机
械截断，从而在控制块大小的同时最大限度保留文本的语义完整性。

                                              图 3.1 递归字符分块法

      该分词方法能够更好地保持语义结构，优先使用段落、换行等更有意义的分

断开，同时它的适应性强，对不同类型的文本结构（段落、列表等）有一定适应

      在文本分块阶段，对几种较为常见的分块技术进行了对比，针对中文文本特
点，基于 RCTS 类提出了改进的 CRTS 类，采用优先级分隔符策略，有效提升了
语义连贯性，实验结果表明，优化后的分块方法能更好地保留中文完整语义单元。
文本向量化环节采用 BGE-Large-zh 模型，该模型在中文文本处理任务中表现较为
出色。在检索阶段，提出了基于加权混合与重排序的信息检索方式，通过加权算
法结合两种检索方式的优势，在专业术语匹配和语义理解方面都取得良好效果。
实验数据显示，混合检索方法的性能指标显著优于单一检索方式。最后对介绍了
本文的提示词模板的构成以及最后生成答案的大语言模型 DeepSeek。

      本章构建了一套完整的高炉领域 RAG 解决方案，为后续系统实现提供了关键
技术支撑，实验证明，各环节优化提升了知识检索的准确性与生成答案的专业性。

      系统响应速度要够快，针对简单问题比如常见的高炉操作规范询问，需在 1

某个组件出现故障之时，像语音识别 API 临时无法使用，系统可自动切换备用方

                                第 5 章 总结与展望
5.1. 论文总结

      本文设计并实现了一套基于大语言模型和 RAG 技术的高炉智能问答系统，此
系统能契合钢铁行业里的智能化需求，把大语言模型的自然语言理解与生成能力、
RAG 技术的外部知识检索能力以及自然语言转 SQL 技术融合起来，该系统可为高
炉操作人员给予精确、及时的专业问答以及数据查询支持，使得高炉操作的效率、
安全性以及智能化水平得到了提升。

      在理论研究层面，本文规划了整个检索增强生成的架构，推出了改进的文本
分块方法（CRTS），让中文长文档的分块效果得到了优化；设计了基于加权混合
与重排序的检索算法，将 BM25 关键词检索和 FAISS 向量检索的优势结合起来，
让检索精度有所提高；依靠构建专业的提示词模板并集成高性能的 DeepSeek 大模
型，系统生成的答案有专业性又有可解释性。

      在系统实现方面，本文运用 LangChain 框架将算法以及模型整合到一起，还
利用 FastAPI 和 Streamlit 完成了智能高炉问答系统的前后端设计，依次开展了问
答系统的功能分析、系统设计、系统实现以及系统测试工作。

5.2. 未来展望

      本文完成了基于大语言模型和 RAG 技术的高炉智能问答系统，但该问答系统
仍有进一步优化的空间，未来可从以下方向展开探索和改进：

      （1）未来可以引入多模态大模型，达成对图像、声音等非结构化数据的联合
分析，如借助炉内图像识别炉况异常，或者结合声音频谱诊断设备故障。

      （2）在检索模块方面，可探索动态混合检索策略，依据问题类型自动调节关
键词以及语义检索权重，引入多阶段检索架构，先进行粗筛再开展精排，以此提
升结果的准确性。

      （3）提示词工程可优化动态模板生成，针对不同问题类型自动匹配最佳
Prompt 结构，结合用户角色设计差异化提示策略，提高回答的针对性。
