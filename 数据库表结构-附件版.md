# 附件：数据库表结构

数据库名：`coding_assistant`  
字符集：`utf8mb4`  
排序规则：`utf8mb4_unicode_ci`

## A. 表清单

1. `user`（用户表）
2. `conversation`（会话表）
3. `message`（消息表）
4. `feedback`（反馈表）
5. `code_snippet`（代码片段表）
6. `learning_path`（学习路径表）
7. `learning_node`（学习路径节点表）
8. `learning_record`（学习行为记录表）
9. `knowledge_document`（知识文档表）
10. `knowledge_chunk`（知识分块表）

---

## 1) user（用户表）

| 字段名 | 类型 | 约束/默认值 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(255) | NOT NULL | 密码（加密） |
| email | VARCHAR(100) | NOT NULL, UNIQUE | 邮箱 |
| avatar | VARCHAR(255) | DEFAULT NULL | 头像地址 |
| level | VARCHAR(20) | DEFAULT 'beginner' | 学习等级 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

索引：
- `idx_user_username(username)`
- `idx_user_email(email)`

---

## 2) conversation（会话表）

| 字段名 | 类型 | 约束/默认值 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 会话ID |
| user_id | BIGINT | NOT NULL, FK | 所属用户ID |
| title | VARCHAR(200) | DEFAULT NULL | 会话标题 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

索引：
- `idx_conversation_user_id(user_id)`

外键：
- `fk_conversation_user`: `conversation.user_id -> user.id`（ON DELETE CASCADE）

---

## 3) message（消息表）

| 字段名 | 类型 | 约束/默认值 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 消息ID |
| conversation_id | BIGINT | NOT NULL, FK | 所属会话ID |
| role | VARCHAR(20) | NOT NULL | 角色（user/assistant） |
| content | TEXT | NOT NULL | 消息内容 |
| sources | JSON | DEFAULT NULL | 来源依据 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |

索引：
- `idx_message_conversation_id(conversation_id)`

外键：
- `fk_message_conversation`: `message.conversation_id -> conversation.id`（ON DELETE CASCADE）

---

## 4) feedback（反馈表）

| 字段名 | 类型 | 约束/默认值 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 反馈ID |
| message_id | BIGINT | NOT NULL, FK | 被反馈消息ID |
| user_id | BIGINT | NOT NULL, FK | 反馈用户ID |
| rating | VARCHAR(10) | NOT NULL | 评分（useful/useless） |
| comment | TEXT | DEFAULT NULL | 反馈内容 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |

唯一约束：
- `uk_feedback_message_user(message_id, user_id)`

外键：
- `fk_feedback_message`: `feedback.message_id -> message.id`（ON DELETE CASCADE）
- `fk_feedback_user`: `feedback.user_id -> user.id`（ON DELETE CASCADE）

---

---

## 7) learning_node（学习路径节点表）

| 字段名 | 类型 | 约束/默认值 | 说明 |

## 8) learning_record（学习行为记录表）

---

## 10) knowledge_chunk（知识分块表）

| 字段名 | 类型 | 约束/默认值 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 分块ID |
| document_id | BIGINT | NOT NULL, FK | 所属文档ID |
| chunk_index | INT | NOT NULL | 文档内分块序号 |
| content | TEXT | NOT NULL | 分块内容 |
| embedding | JSON | DEFAULT NULL | 分块向量 |

- `uk_chunk_doc_idx(document_id, chunk_index)`

- `idx_chunk_document_id(document_id)`

外键：
- `fk_chunk_document`: `knowledge_chunk.document_id -> knowledge_document.id`（ON DELETE CASCADE）

---

## B. 关系说明（论文可用）

1. `user` 是核心主表，和 `conversation`、`feedback`、`code_snippet`、`learning_path`、`learning_record` 存在一对多关系。  
2. `conversation` 和 `message` 为一对多，会话删除时消息级联删除。  
3. `message` 和 `feedback` 为一对多（在用户维度下通过联合唯一约束限制同一用户对同一消息仅一条反馈）。  
4. `learning_path` 和 `learning_node` 为一对多，路径删除时节点级联删除。  
5. `knowledge_document` 和 `knowledge_chunk` 为一对多，文档删除时分块级联删除。
