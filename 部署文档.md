## 一、项目概述

**智能编程学习助手** 是一个基于 Vue3 + Spring Boot + Neo4j 知识图谱 + LLM 大模型 的全栈应用，包含以下功能模块：

| 模块 | 功能说明 |
|------|----------|
| 智能问答 | 基于 LLM 的编程问答，支持流式输出、多轮对话、代码高亮 |
| 知识图谱 | 基于 Neo4j 的编程知识可视化，支持搜索、展开、筛选 |
| 学习路径 | AI 生成个性化学习路线，支持状态跟踪 |
| 代码片段 | 代码收藏管理，支持搜索、标签、导入导出 |
| 学习仪表盘 | 学习统计、活跃度热力图、知识雷达图、PDF 报告导出 |
| 用户系统 | 注册登录、JWT 认证、头像上传、个人中心 |

### 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + TypeScript + Vite + Element Plus + ECharts + Monaco Editor |
| 后端 | Spring Boot 3.2.5 + Java 21 + MyBatis-Plus + Spring Security |
| 数据库 | MySQL 8.0 + Neo4j 5 |
| AI | DeepSeek / OpenAI / ChatGLM / Qwen（可切换） |
| 部署 | Docker + Docker Compose + Nginx |

## 三、方式一：Docker Compose 一键部署（推荐）

### 3.1 获取代码

```bash
# 将项目文件夹复制到部署服务器
# 或通过 Git 克隆（如有仓库）
cd intelligent-coding-assistant
```

### 3.2 配置环境变量

复制环境变量模板并修改：

```bash
cp .env.example .env    # 如果没有 .env.example，直接编辑 .env
```

编辑 `.env` 文件，**必须修改以下配置**：

```ini
# ============================================================
# MySQL 配置
# ============================================================
MYSQL_ROOT_PASSWORD=your_mysql_password    # 修改为安全密码
MYSQL_DATABASE=coding_assistant

# ============================================================
# Neo4j 配置
# ============================================================
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_neo4j_password         # 修改为安全密码

# ============================================================
# JWT 密钥（至少32位随机字符串）
# ============================================================
JWT_SECRET=your_jwt_secret_key_at_least_32_characters_long

# ============================================================
# LLM 大模型配置（必须配置才能使用智能问答功能）
# ============================================================
LLM_PROVIDER=openai
OPENAI_API_KEY=your-api-key-here           # 填入 API Key
OPENAI_BASE_URL=https://api.deepseek.com   # DeepSeek 接口地址
OPENAI_MODEL=deepseek-chat                 # 模型名称
```

**支持的 LLM 提供商：**

| 提供商 | BASE_URL | MODEL | 说明 |
|--------|----------|-------|------|
| DeepSeek | https://api.deepseek.com | deepseek-chat | 推荐，性价比高 |
| OpenAI | https://api.openai.com | gpt-4 / gpt-3.5-turbo | 需要海外网络 |
| 月之暗面 Kimi | https://api.moonshot.cn | moonshot-v1-8k | 国内可用 |

### 3.3 启动所有服务

```bash
docker-compose up -d
```

首次启动会自动完成以下操作：
1. 拉取 MySQL 8.0、Neo4j 5、Nginx 镜像
2. 构建后端 Spring Boot JAR（约 3-5 分钟）
3. 构建前端 Vue 3 静态文件（约 1-2 分钟）
4. 初始化 MySQL 数据库表结构和测试数据
5. 启动所有服务

### 3.4 初始化 Neo4j 知识图谱数据

Neo4j 知识图谱数据会在**后端应用首次启动时自动初始化**（通过 `Neo4jDataInitializer` 组件）。

如需手动导入更完整的数据：

```bash
# 等待 Neo4j 完全启动后执行
cat docker/neo4j/init-data.cypher | docker exec -i ica-neo4j cypher-shell -u neo4j -p your_neo4j_password
```

### 3.5 验证启动成功

```bash
# 查看所有容器状态
docker-compose ps
```

预期输出（所有服务状态应为 `Up`）：

```
NAME              STATUS          PORTS
ica-mysql         Up (healthy)    0.0.0.0:3306->3306/tcp
ica-neo4j         Up (healthy)    0.0.0.0:7474->7474/tcp, 0.0.0.0:7687->7687/tcp
ica-backend       Up (healthy)    8080/tcp
ica-nginx         Up              0.0.0.0:80->80/tcp
ica-frontend-build  Exited (0)    （正常，构建完成即退出）
```

### 3.6 访问系统

| 服务 | 地址 | 说明 |
|------|------|------|
| **系统首页** | http://localhost | 主入口，Nginx 托管 |
| Neo4j 浏览器 | http://localhost:7474 | 知识图谱管理界面 |

### 3.7 常用 Docker 命令

```bash
# 查看日志
docker-compose logs -f backend     # 后端日志
docker-compose logs -f mysql       # MySQL 日志
docker-compose logs -f neo4j       # Neo4j 日志

# 停止所有服务
docker-compose down

# 停止并清除数据卷（重置所有数据）
docker-compose down -v

# 重新构建后端（代码更新后）
docker-compose build backend
docker-compose up -d backend

# 重新构建前端（代码更新后）
docker-compose build frontend
docker-compose up -d frontend nginx

# 进入 MySQL 命令行
docker exec -it ica-mysql mysql -uroot -p

# 进入后端容器
docker exec -it ica-backend sh
```

## 四、方式二：本地开发环境部署

适用于开发调试或无 Docker 环境的场景。

### 4.1 安装基础软件

#### 4.1.1 安装 JDK 21

**macOS：**
```bash
brew install openjdk@21
# 配置环境变量
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 21)' >> ~/.zshrc
source ~/.zshrc
java -version   # 验证
```

**Windows：**
1. 下载 [Eclipse Temurin JDK 21](https://adoptium.net/)
2. 安装时勾选"设置 JAVA_HOME"
3. 打开新命令行验证：`java -version`

**Linux (Ubuntu/Debian)：**
```bash
sudo apt install openjdk-21-jdk
java -version
```

#### 4.1.2 安装 Maven

**macOS：**
```bash
brew install maven
mvn -version    # 验证
```

**Windows：**
1. 下载 [Maven](https://maven.apache.org/download.cgi)
2. 解压到 `C:\Program Files\Maven`
3. 添加 `C:\Program Files\Maven\bin` 到系统 PATH
4. 验证：`mvn -version`

**Linux：**
```bash
sudo apt install maven
mvn -version
```

#### 4.1.3 安装 Node.js

**macOS：**
```bash
brew install node@20
node -v    # 验证（需要 20.19+ 或 22.12+）
npm -v
```

**Windows / Linux：**
1. 下载 [Node.js LTS](https://nodejs.org/)
2. 安装并验证：`node -v` 和 `npm -v`

#### 4.1.4 安装 MySQL 8.0

**macOS：**
```bash
brew install mysql@8.0
brew services start mysql@8.0
```

**Windows：**
1. 下载 [MySQL Installer](https://dev.mysql.com/downloads/installer/)
2. 选择 "MySQL Server 8.0"，按向导安装
3. 记住设置的 root 密码

**Linux (Ubuntu/Debian)：**
```bash
sudo apt install mysql-server-8.0
sudo systemctl start mysql
sudo mysql_secure_installation
```

#### 4.1.5 安装 Neo4j 5

**macOS：**
```bash
brew install neo4j
neo4j start
```

**Windows / Linux：**
1. 下载 [Neo4j Community Edition](https://neo4j.com/download/)
2. 解压后执行 `bin/neo4j start`
3. 首次访问 http://localhost:7474 ，默认账号 `neo4j/neo4j`，会要求修改密码

### 4.2 初始化数据库

#### 4.2.1 MySQL 初始化

```bash
# 登录 MySQL
mysql -u root -p

# 执行初始化脚本（创建数据库、表结构、测试数据）
source /path/to/intelligent-coding-assistant/backend/src/main/resources/schema.sql
```

或：

```bash
mysql -u root -p < backend/src/main/resources/schema.sql
```

#### 4.2.2 Neo4j 初始化

Neo4j 知识图谱数据会在后端应用首次启动时**自动初始化**，无需手动操作。

如需手动导入完整数据：

```bash
# 方式一：通过 cypher-shell（需安装 Neo4j 命令行工具）
cat docker/neo4j/init-data.cypher | cypher-shell -u neo4j -p neo4j123

# 方式二：通过 Neo4j Browser
# 打开 http://localhost:7474
# 将 docker/neo4j/init-data.cypher 内容分段粘贴执行
```

### 4.3 配置后端

编辑 `backend/src/main/resources/application.yml`，确认以下配置：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/coding_assistant?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai
    username: root                  # 你的 MySQL 用户名
    password: your_mysql_password   # 你的 MySQL 密码

neo4j:
  uri: bolt://localhost:7687
  username: neo4j
  password: neo4j123               # 你的 Neo4j 密码

llm:
  provider: openai
  openai:
    api-key: your-api-key          # 你的 LLM API Key
    base-url: https://api.deepseek.com
    model: deepseek-chat
```

也可以通过环境变量覆盖（推荐生产环境使用）：

```bash
export MYSQL_PASSWORD=your_password
export NEO4J_PASSWORD=neo4j123
export OPENAI_API_KEY=your-api-key
export OPENAI_BASE_URL=https://api.deepseek.com
export OPENAI_MODEL=deepseek-chat
```

### 4.4 启动后端

```bash
cd backend

# 方式一：Maven 命令运行（开发模式）
mvn spring-boot:run

# 方式二：先编译再运行
mvn clean package -DskipTests
java -jar target/coding-assistant-1.0.0-SNAPSHOT.jar
```

启动成功标志：
```
Started CodingAssistantApplication in X.XX seconds
```

后端运行在 http://localhost:8080

### 4.5 启动前端

打开**新的终端窗口**：

```bash
cd frontend

# 安装依赖（首次或依赖变更时执行）
npm install

# 启动开发服务器
npm run dev
```

启动成功标志：
```
VITE vX.X.X  ready in XXXms
  ➜  Local:   http://localhost:5173/
```

前端运行在 http://localhost:5173

### 4.6 访问系统

打开浏览器访问 **http://localhost:5173**

> 注意：本地开发模式下，前端通过 Vite 代理自动将 `/api` 请求转发到 `http://localhost:8080`，无需额外配置 Nginx。

### 4.7 前端生产构建（可选）

如果需要部署到生产服务器：

```bash
cd frontend
npm run build
```

构建产物输出在 `frontend/dist/` 目录，可部署到任意静态文件服务器（Nginx、Apache 等）。

配合 Nginx 使用时，参考 `docker/nginx/nginx.conf` 配置：
- 静态文件 → `frontend/dist/`
- `/api/` 反向代理 → `http://backend-server:8080/api/`
- `/uploads/` 反向代理 → `http://backend-server:8080/uploads/`（头像等上传文件）

## 六、常见问题与排查

### 6.1 Docker 部署问题

**Q：`docker-compose up` 构建失败**
```bash
# 清除缓存重新构建
docker-compose build --no-cache
docker-compose up -d
```

**Q：后端启动失败，提示连接 MySQL 超时**
```bash
# 检查 MySQL 是否启动完成
docker-compose logs mysql
# 等待 healthy 状态后重启后端
docker-compose restart backend
```

**Q：Neo4j 知识图谱页面空白**
```bash
# 手动导入数据
cat docker/neo4j/init-data.cypher | docker exec -i ica-neo4j cypher-shell -u neo4j -p your_password
# 重启后端让初始化逻辑重新检测
docker-compose restart backend
```

**Q：端口冲突（80/3306/7474/7687 被占用）**
```bash
# 查看端口占用
lsof -i :80        # macOS/Linux
netstat -ano | findstr :80  # Windows

# 修改 docker-compose.yml 中的端口映射
# 例如将 "80:80" 改为 "8888:80"
```

### 6.2 本地部署问题

**Q：`mvn spring-boot:run` 报错 "找不到 JDK 21"**
```bash
# 确认 JAVA_HOME 指向 JDK 21
echo $JAVA_HOME
java -version

# macOS 设置
export JAVA_HOME=$(/usr/libexec/java_home -v 21)
```

**Q：`npm install` 很慢或失败**
```bash
# 使用国内镜像
npm config set registry https://registry.npmmirror.com
npm install
```

**Q：前端页面空白 / API 请求 404**
- 确认后端已启动并监听 8080 端口
- 确认前端 `vite.config.ts` 中 proxy 配置正确
- 检查浏览器控制台和网络请求面板

**Q：智能问答没有回复 / 超时**
- 确认 LLM API Key 有效
- 确认 API Base URL 正确
- 检查后端日志：`docker-compose logs backend` 或终端输出
- DeepSeek API 偶尔有延迟，可等待或重试

**Q：头像上传失败**
- 确认后端 `uploads/` 目录有写入权限
- 图片大小不超过 5MB
- 支持格式：JPG、PNG、GIF、WebP

### 6.3 数据重置

```bash
# Docker 方式 - 完全重置
docker-compose down -v
docker-compose up -d

# 本地方式 - 重新导入 MySQL 数据
mysql -u root -p < backend/src/main/resources/schema.sql
```

### 8.7 通用说明

- **认证方式**：在请求头中携带 `Authorization: Bearer <token>`
- **响应格式**：所有接口统一返回 JSON 格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

- **错误码**：

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
